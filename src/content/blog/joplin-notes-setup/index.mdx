## Docker Compose Configuration

I'm running Joplin Server alongside my other homelab services. Here's the complete docker-compose setup:

```yaml
services:
  # ... your other services ...

  joplin-db:
    image: postgres:15
    container_name: joplin-db
    restart: unless-stopped
    environment:
      POSTGRES_PASSWORD: your_secure_password_here  # Change this!
      POSTGRES_USER: joplin
      POSTGRES_DB: joplin
    volumes:
      - joplin_db_data:/var/lib/postgresql/data
    networks:
      network-cloud-merox:
        ipv4_address: 172.25.10.60

  joplin-server:
    image: joplin/server:latest
    container_name: joplin-server
    restart: unless-stopped
    depends_on:
      - joplin-db
    dns:
      - 1.1.1.1
      - 8.8.8.8
    environment:
      APP_PORT: 22300
      APP_BASE_URL: https://joplin.cloud.merox.dev  # Your domain here
      DB_CLIENT: pg
      POSTGRES_PASSWORD: your_secure_password_here  # Same as above
      POSTGRES_DATABASE: joplin
      POSTGRES_USER: joplin
      POSTGRES_PORT: 5432
      POSTGRES_HOST: joplin-db
    networks:
      network-cloud-merox:
        ipv4_address: 172.25.10.61
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.joplin.rule=Host(`joplin.cloud.merox.dev`)"
      - "traefik.http.routers.joplin.entrypoints=https"
      - "traefik.http.routers.joplin.tls.certresolver=cloudflare"
      - "traefik.http.services.joplin.loadbalancer.server.port=22300"
      - "traefik.docker.network=network-cloud-merox"

volumes:
  joplin_db_data:

networks:
  network-cloud-merox:
    name: network-cloud-merox
    driver: bridge
    ipam:
      config:
        - subnet: 172.25.0.0/16
          gateway: 172.25.0.1
```

<Callout variant="tip">
**Network Setup:** I'm using a custom bridge network with static IPs for better service discovery. Adjust the subnet and IPs to match your existing infrastructure.
</Callout>

### Key Configuration Points

**Database Choice:** I went with PostgreSQL instead of SQLite for better reliability and concurrent access. If you're running a single-user setup, SQLite works fine too.

**Environment Variables:**
- `APP_BASE_URL` - Your public HTTPS URL (critical for correct redirects)
- `POSTGRES_PASSWORD` - Use a strong password, stored securely
- `APP_PORT` - Internal port (22300 is Joplin's default)

**Traefik Labels:**
- Automatic HTTPS via Cloudflare
- Host-based routing
- Load balancer pointing to internal port 22300

## DNS Configuration

Since I'm using PiHole for local DNS resolution, I added the record there instead of only in Cloudflare:

### Option 1: PiHole Local DNS (if you run PiHole)

Edit `./etc-dnsmasq.d/04-custom.conf`:

```bash
address=/joplin.cloud.merox.dev/100.72.22.38
```

Then restart PiHole:
```bash
docker restart pihole
```

### Option 2: Cloudflare DNS Only

Add an A record in Cloudflare:
- **Type:** A
- **Name:** joplin.cloud (or just joplin if your subdomain is different)
- **Content:** Your server's public IP
- **Proxy:** DNS only (orange cloud off initially)

<Callout variant="note">
**Why both?** PiHole handles internal resolution via my Tailscale IP (100.72.22.38), while Cloudflare handles external access. You can use either method depending on your setup.
</Callout>

## Deployment

```bash
# Navigate to your docker-compose directory
cd /path/to/your/docker-compose

# Pull the images
docker-compose pull joplin-db joplin-server

# Start the services
docker-compose up -d joplin-db joplin-server

# Watch the logs
docker logs -f joplin-server
```

You should see output like:
```
Server started on port 22300
Database connection successful
```

## Common Issues and Fixes

### Sync Fails with "Connection Refused"

**Problem:** Mobile apps can't connect to server.

**Fix:** Check that your domain resolves correctly:
```bash
# Test DNS resolution
nslookup joplin.cloud.merox.dev

# Test HTTPS connectivity
curl -I https://joplin.cloud.merox.dev
```

If DNS fails, verify your PiHole/Cloudflare records. If HTTPS fails, check Traefik logs:
```bash
docker logs traefik
```

### "Invalid Email or Password"

**Problem:** Can't log in to server.

**Fix:** 
1. Verify you're using the sync user account (not admin@localhost)
2. Double-check the URL doesn't have trailing slashes
3. Ensure APP_BASE_URL in docker-compose matches exactly

### Slow Sync on Mobile

**Problem:** Sync takes forever on iOS/Android.

**Fix:**
- First sync is always slow (downloading all notes)
- Check network connectivity (WiFi vs cellular)
- Verify server isn't rate limiting (check `docker logs joplin-server`)
- Consider enabling E2E encryption for faster sync of large files

### Database Connection Errors

**Problem:** Server logs show "Cannot connect to database"

**Fix:**
```bash
# Check if PostgreSQL is running
docker ps | grep joplin-db

# Check database logs
docker logs joplin-db

# Verify password matches in both containers
docker exec joplin-server env | grep POSTGRES
docker exec joplin-db env | grep POSTGRES
```

<Callout variant="tip">
**Pro Tip:** Enable auto-sync in mobile apps (Settings â†’ Synchronisation â†’ Synchronisation interval). Set to 5-10 minutes for near-real-time sync.
</Callout>

## Backup Strategy

Since you control the infrastructure, you control the backups. Here's my setup:

```bash
# Backup PostgreSQL database
docker exec joplin-db pg_dump -U joplin joplin > joplin-backup-$(date +%Y%m%d).sql

# Automate with cron
0 2 * * * docker exec joplin-db pg_dump -U joplin joplin > /backups/joplin-$(date +\%Y\%m\%d).sql
```

**Screenshot placeholder:** `joplin-backup-script.png` - Terminal showing backup script running and creating dated backup files

For extra safety, I sync these backups to my NAS and occasionally to cloud storage. The beauty of self-hosting is you define your own redundancy.

<Callout variant="note">
**Database Backups:** If you're already running Longhorn in Kubernetes (like I am), you could also deploy Joplin there and leverage automatic PVC snapshots. But Docker Compose is simpler for this use case.
</Callout>
